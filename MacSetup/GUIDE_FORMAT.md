# MacSetup Guide Markdown Format

이 문서는 MacSetup 앱에서 읽기 좋은 Markdown 가이드를 작성하기 위한 약속입니다. 프로젝트가 바뀌어도 이 규칙대로 `.md` 파일만 바꾸면 사이드바 섹션, 체크 카드, 명령 카드, 안내칸이 자동으로 만들어집니다.

앱에는 기본 Markdown 가이드가 포함되지 않습니다. 완성한 `.md` 파일을 앱 창에 드래그앤드롭하거나 `+` 버튼으로 추가해서 사용합니다.

## 1. 파일 상단 헤더

파일 맨 위에 `---`로 감싼 메타데이터를 둡니다. 앱은 이 값을 가이드 요약 영역에 표시합니다.

```md
---
title: ProjectName 신규 Mac 세팅
summary: 이 가이드가 끝나면 클론, 의존성 설치, 빌드, 실행 확인까지 완료됩니다.
version: 1.0
owner: Team Name
updated: 2026-07-09
estimated: 30-45분
---
```

지원 키:
- `title`: 앱에 표시될 가이드 이름
- `summary`: 가이드 상단 설명
- `version`: 가이드 버전
- `owner`: 담당 팀 또는 담당자
- `updated`: 마지막 업데이트 날짜
- `estimated`: 예상 소요 시간

## 2. 카드 구조

````md
# 문서 제목

## 1. 큰 단계

### 체크 가능한 작업 카드

설명 문장입니다. 일반 Markdown의 `code`, **bold**를 사용할 수 있습니다.

```bash
echo "명령어 카드"
```
````

규칙:
- `#` H1은 문서 제목입니다.
- `##` H2는 왼쪽 사이드바 섹션입니다.
- `###` H3는 체크 가능한 작업 카드입니다.
- 코드블록은 실행/복사 가능한 명령 카드입니다.
- 코드블록 언어는 되도록 `bash`를 씁니다.

## 3. 작성자 주석

앱에 보여주고 싶지 않은 작성자용 메모는 Markdown 주석으로 씁니다.

```md
<!--
이 내용은 앱 카드에 표시되지 않습니다.
다음 프로젝트 때 명령어만 바꾸면 됩니다.
-->
```

## 4. 안내칸

GitHub 스타일 콜아웃을 씁니다.

```md
> [!NOTE]
> 일반 안내입니다.

> [!TIP]
> 더 편한 방법이나 선택지를 적습니다.

> [!IMPORTANT]
> 반드시 확인해야 하는 내용입니다.

> [!WARNING]
> 실수하기 쉬운 내용입니다.

> [!DANGER]
> 그대로 실행하면 위험하거나 되돌리기 어려운 내용입니다.
```

앱 표시 색상:
- `NOTE`: 파란 안내
- `TIP`: 초록 팁
- `IMPORTANT`: 보라색 중요
- `WARNING`: 주황색 주의
- `DANGER`: 빨간색 위험

## 5. 빨간 굵은 강조

문장 안에서 반드시 눈에 띄어야 하는 부분은 앱 전용 토큰을 씁니다.

```md
`YOUR_EMAIL`은 {{red:반드시 본인 회사 이메일로 교체}}한 뒤 실행합니다.
```

이 토큰은 앱에서 빨간색 굵은 글씨로 표시됩니다. 일반 Markdown 뷰어에서는 그대로 보일 수 있으므로 남발하지 말고 정말 위험하거나 중요한 부분에만 씁니다.

## 6. 실행 안전 규칙

앱은 다음 패턴을 안전하게 다룹니다.

- `YOUR_`가 포함된 명령어는 앱 내 실행과 Terminal 실행이 막힙니다. 먼저 복사해서 개인 값으로 수정해야 합니다.
- `claude login`, `ssh -T`, `git clone`처럼 인증이나 긴 작업이 섞인 명령어는 앱 내 실행보다 Terminal 실행을 권장합니다.
- `sudo`가 포함된 명령어는 관리자 권한 대화상자를 띄워 실행합니다.

## 7. 권장 작성 순서

1. 시작 전 준비물
2. 필수 도구 설치
3. 계정/권한 설정
4. 프로젝트 클론
5. 의존성 설치
6. 로컬 전용 파일 배치
7. 빌드/실행
8. 권한 확인
9. 트러블슈팅

## 8. 짧은 템플릿

````md
---
title: ProjectName 개발환경 세팅
summary: 새 Mac에서 ProjectName을 실행할 수 있게 만드는 가이드입니다.
version: 1.0
owner: Team Name
updated: 2026-07-09
estimated: 30분
---

# ProjectName 개발환경 세팅

## 0. 시작 전 확인

### 계정과 권한 확인

> [!IMPORTANT]
> GitHub 저장소 접근 권한이 먼저 필요합니다.

- GitHub 계정
- 저장소 접근 권한
- 로컬 설정 파일

## 1. 프로젝트 준비

### 프로젝트 클론

```bash
cd ~/dev
git clone git@github.com:ORG/PROJECT.git
cd PROJECT
```

### 개인 설정 입력

> [!DANGER]
> `YOUR_EMAIL`은 {{red:본인 회사 이메일로 교체}}합니다.

```bash
git config user.email "YOUR_EMAIL"
```
````
